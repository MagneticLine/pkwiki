# 聊天记录：pkwiki 项目规划讨论

> 日期：2026-07-03
> 参与者：开发者 / AI 助手
> 话题：pkwiki 低敏知识库系统设计

---

## 对话记录

**开发者**：我想搭一个个人知识库系统，叫 pkwiki。核心需求是把日常碎片化的 AI 对话、项目笔记、学习片段自动整理成结构化的 wiki 页面。你有什么建议？

**AI 助手**：这个方向很实用。我可以从几个维度帮你梳理一下：

1. **数据源接入**：AI 聊天记录、浏览器书签、读书笔记截图 OCR、手动 markdown 文件。建议先做 markdown 文件 + 聊天记录解析，其他作为后续扩展。
2. **页面类型**：建议设计 5 种核心页面类型 —— Project（项目）、Decision（决策记录）、LearningMap（学习路径）、CareerMaterial（个人资料）、DailyNote（每日笔记）。
3. **自动化程度**：分阶段 —— 第一阶段手动触发整理、第二阶段定时自动扫描、第三阶段结合 LLM 自动分类。

**开发者**：页面类型这个分类我很认同。能展开说说每个页面应该包含哪些字段吗？

**AI 助手**：我的建议：

- **Project**：项目名、目标、当前阶段、关联决策列表、关键学习记录链接、更新时间
- **Decision**：决策标题、背景、选项对比、最终选择、决策日期、后续影响
- **LearningMap**：主题名称、前置知识、核心概念列表、实践练习、掌握程度、关联项目
- **CareerMaterial**：类型（简历/技能清单/成果展示）、摘要标签、有效期、可公开范围
- **DailyNote**：日期、标签、核心事件、关联页面引用

**开发者**：关于低敏和安全，我想确保整个系统设计上就避免收集敏感信息。有什么清单可以参考？

**AI 助手**：低敏设计清单建议包含以下原则：

- **绝对不收集**：身份证号、手机号、住址、银行卡、健康档案、家庭成员信息、公司内部代码/密钥
- **建议脱敏**：公司名称可用行业描述替代、项目名可用代号、公共场所出现的人名可用角色名替代
- **公开度分级**：public（完全可分享）、internal（仅自己可见）、draft（草稿/未完成可公开）
- **存储原则**：所有内容本地存储、不上传云端除非用户明确操作；LLM 调用时只发送当前页面的摘要级内容

**开发者**：好，有了这些原则我可以开始搭建最小版本了。下个阶段先把聊天记录的解析和 wiki 更新流程跑通。

**AI 助手**：建议第一阶段的技术选型：

- 文件存储用本地 markdown + frontmatter（YAML 元数据）
- 索引用 SQLite，轻量且无需额外部署
- LLM 调用量不大时直接用本地模型（如 Ollama 跑的 7B 模型），敏感数据不出本机
- 解析器用 Node.js，方便复用前端工具链
