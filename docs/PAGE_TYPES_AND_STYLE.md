# Page Types And Style

个人 Wiki 不是单一类型的百科，也不是纯日记。它需要用页面类型区分不同信息的落位方式。

页面类型和写作风格是 Agent 判断“素材应该写到哪里、写多细、保留什么、舍弃什么”的核心依据。

## 1. 页面类型原则

- 页面类型服务 Source-to-Wiki Merge，不只是 frontmatter 标签。
- 不同页面类型允许不同的时间粒度、隐私等级、证据要求和更新方式。
- Agent 不确定页面类型时，应先记录候选类型并请求确认。

## 2. 基础页面类型

### 2.1 `Profile`

长期个人画像。

适合：

- 稳定身份信息。
- 长期偏好。
- 性格和行为模式。
- 能帮助 Agent 理解 Human Maintainer 的背景信息。

不适合：

- 每日流水账。
- 未确认的心理推断。

### 2.2 `LifeEvent`

个人经历事件。

适合：

- 入学、毕业、实习、工作、搬家、重大关系变化。
- 明确时间点或时间段的经历。
- 对未来判断有影响的节点。

### 2.3 `Project`

项目和长期任务。

适合：

- 软件项目。
- 申请项目。
- 学习项目。
- 职业项目。

### 2.4 `CareerMaterial`

职业和申请材料。

适合：

- 简历。
- 推荐信素材。
- 申请文书。
- 作品集材料。
- 证书、成绩单、项目证明。

这类页面应重点保留原始材料路径和版本线索。

### 2.5 `Reflection`

反思和思想演化。

适合：

- 日记中的总结。
- 对失败和选择的复盘。
- 对自我认知的变化。
- 价值观和目标变化。

规则：

- 必须区分当时想法和当前结论。
- 不应把一次情绪表达固化成长期人格判断。

### 2.6 `Decision`

明确决策记录。

适合：

- 做过的关键选择。
- 选择理由。
- 当时约束。
- 结果和后续复盘。

### 2.7 `HealthRecord`

健康相关记录。

适合：

- 基因检测报告。
- 体检报告。
- 长期健康指标。
- 医疗建议摘要。

规则：

- 默认 private。
- 不进行医疗结论推断。
- 不确定时保留来源和问题，不写成确定建议。

### 2.8 `PsychologicalPattern`

心理和行为模式。

适合：

- 反复出现的情绪、动机、拖延、压力模式。
- 用户自己确认过的自我剖析。

规则：

- 高敏感。
- 需要来源和置信度。
- 避免诊断式语言。

### 2.9 `LearningNote`

课程、书籍、论文、技术资料学习记录。

适合：

- 课程资料整理。
- 学习路线。
- 关键概念。
- 练习和项目成果。

### 2.10 `Concept`

概念页。

适合：

- 可复用知识概念。
- 技术、产品、思想、理论。
- 外部知识而非个人经历。

### 2.11 `Timeline`

时间线。

适合：

- 跨多个页面的重要事件索引。
- 项目时间线。
- 人生阶段时间线。

### 2.12 `SourceSummary`

素材摘要页。

适合：

- 大型 source 的长期入口。
- 不适合拆成多个长期 Wiki Page 的 source。

注意：SourceSummary 不替代 Extracted Source。

## 3. 写作风格

Wiki 写作应追求：

- 清晰。
- 可追溯。
- 克制。
- 长期可维护。
- 对未来 Agent 有用。

不追求：

- 华丽文风。
- 情绪强化。
- 过度总结人格。
- 把短期想法写成长期结论。

## 4. 信息粒度

Agent 合并素材时应优先保留：

- 能改变对 Human Maintainer 理解的信息。
- 会影响未来建议和决策的信息。
- 具有长期复用价值的材料。
- 与已有页面冲突或补充的信息。
- Human Maintainer 明确要求保留的信息。

可以 deferred 的内容：

- 细碎、无长期价值的聊天噪音。
- 无法判断重要性的上下文。
- 缺少来源或含义不明的信息。

可以 discarded 的内容：

- 明显重复且已有更好记录。
- 纯格式噪音。
- 与 Wiki 目标无关且无长期价值的信息。

Discarded 必须记录原因。

## 5. 用户偏好学习

Agent 的风格应通过版本化规则逐步学习，而不是依赖单次上下文。

建议在 `system/` 中维护：

```text
STYLE_GUIDE.md
MERGE_POLICY.md
PAGE_TYPES.md
PRIVACY_RULES.md
```

当 Human Maintainer 指出 Agent 的合并方式不符合预期时，应优先把可复用偏好沉淀到这些文件，而不是只修一次页面。

## 6. Agent 决策提示

Agent 面对新素材时，应先判断：

1. 这是个人经历、外部知识、可复用材料，还是临时工作记录？
2. 它应该创建新页面、更新旧页面、进入时间线，还是只留在 Extracted Source？
3. 它是否涉及高敏隐私？
4. 它是否改变已有页面的长期结论？
5. 它是否需要用户确认？

如果这些问题无法回答，Agent 应选择 `needs_confirmation`，而不是直接写入 Wiki。
